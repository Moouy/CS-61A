## 项目 2：从 Instagram 账户下载图片

### 目标

在这个项目中，您将下载来自特定公共 Instagram 账号的所有图片，例如 Yan 的账号：[https://www.instagram.com/grapeot/。](https://www.instagram.com/grapeot/%E3%80%82)

### 背景和上下文

与许多社交媒体平台一样，Instagram 对自动化数据提取施加了严格限制。申请 API 密钥通常较为耗时，平台也采用了反机器人措施来防止未经授权的抓取。页面内容还是动态加载的，因此并不适合直接用传统方式抓取。不过，借助开发者工具与 AI 辅助，可以合乎道德且高效地从公共账号下载图片。

本项目建立在模块 1 中 GUI 到 API 转换的概念之上，也延续了模块 2 中关于 GenAI 内部机制与最佳实践的内容。它展示了 AI 辅助编程与有效提示管理在解决实际障碍时的作用。

### 这个项目的重要性

自动化获取受限制平台上的数据是一项很有价值的能力。它体现了在遵守道德规范前提下，创造性地利用现有工具达成目标的能力。这个项目也说明了理解 Web 技术和开发者工具在数据提取任务中的实际价值。

### 预期成果

应当能够从指定的 Instagram 账户下载所有公开可用的图片并保存到本地。这通常包括整理图片 URL 列表，并编写脚本自动完成下载。

### 学习目标

学习如何合乎道德地处理平台限制。  
获得使用浏览器开发者工具进行网络分析的实践经验。  
了解如何处理动态加载的内容。  
培养借助 AI 辅助编程完成复杂自动化任务的能力。

### 提示与指导

先在浏览器中打开目标公共 Instagram 账号。虽然页面上的图片是以 GUI 形式动态呈现的，但其底层数据仍然可以被观察到。可以使用浏览器开发者工具，尤其是“Network（网络）”面板，在滚动页面时监控加载的资源，这些资源中就包括图片本身。

Instagram 使用了无限滚动和动态内容加载机制，因此页面内容不会一次性全部加载完成。为了捕获所有图片，需要持续向下滚动页面，让更多内容逐步加载出来。在这个过程中，网络请求记录中会出现对应图片的 URL。

当获得图片 URL 列表后，可以借助 AI 生成下载脚本。向 AI 清楚说明当前遇到的问题，例如动态加载内容、反机器人限制等，有助于获得更准确的脚本和方案。
