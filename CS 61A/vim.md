## Vim 高效快捷键梯度表

### 第一梯度：必须先掌握

这部分覆盖最常见的移动、进入编辑、基础修改与查找，日常使用频率最高。

| 类别  | 快捷键             | 含义          | 高频场景      | 记忆方法         |
| --- | --------------- | -----------:| --------- | ------------ |
| 移动  | `h` `j` `k` `l` | 左 下 上 右     | 基础移动      | 把它当成方向键替代    |
| 移动  | `w`             | 跳到下一个单词开头   | 按词阅读、按词修改 | `w = word`   |
| 移动  | `b`             | 跳到上一个单词开头   | 回退查看上一个词  | `b = back`   |
| 移动  | `0`             | 跳到行首        | 回到本行最左侧   | 数字零像起点       |
| 移动  | `^`             | 跳到行首第一个非空字符 | 代码缩进后定位   | 比 `0` 更贴近正文  |
| 移动  | `$`             | 跳到行尾        | 补内容、加注释   | `$` 常联想到末尾   |
| 编辑  | `i`             | 在光标前进入插入    | 原地输入      | `i = insert` |
| 编辑  | `a`             | 在光标后进入插入    | 接着当前字符写   | `a = append` |
| 编辑  | `o`             | 下方新开一行并编辑   | 写下一行      | 像在下面打开一行     |
| 编辑  | `A`             | 跳到行尾并编辑     | 行尾续写      | 大写版 `a`，作用更大 |
| 删除  | `x`             | 删除当前字符      | 删错字、符号    | `x` 像划掉      |
| 删除  | `dd`            | 删除整行        | 清掉整行代码    | 同类动作重复两次表示整行 |
| 复制  | `yy`            | 复制整行        | 复制一行模板    | `y = yank`   |
| 粘贴  | `p`             | 粘贴到后面       | 复制后放置     | `p = put`    |
| 撤销  | `u`             | 撤销          | 回退操作      | 单键高频         |
| 重做  | `Ctrl-r`        | 重做          | 撤销后恢复     | 与很多软件习惯接近    |
| 查找  | `/文本`           | 向下搜索        | 找变量、函数名   | `/` 像向前查     |
| 查找  | `n`             | 下一个搜索结果     | 连续跳转匹配项   | `n = next`   |

### 第二梯度：开始明显提速

这部分是 Vim 真正拉开效率差距的核心，重点在“操作符 + 范围”。

| 类别   | 快捷键     | 含义            | 高频场景     | 记忆方法                    |
| ---- | ------- | -------------:| -------- | ----------------------- |
| 删除   | `dw`    | 删除到下一个单词开头    | 删除一个词    | `d = delete`，`w = word` |
| 修改   | `cw`    | 改写一个单词        | 改变量名、字段名 | `c = change`            |
| 删除   | `d$`    | 删除到行尾         | 清掉本行后半段  | `$ = 行尾`                |
| 修改   | `c$`    | 改写到行尾         | 重写后半行    | `c + 行尾`                |
| 修改   | `cc`    | 改写整行          | 整行重写     | 双击同键表示整行                |
| 跳转   | `gg`    | 到文件开头         | 快速回顶部    | 双 `g` 表示全局起点            |
| 跳转   | `G`     | 到文件结尾         | 快速到底部    | 大写通常表示更大范围              |
| 跳转   | `nG`    | 到指定行          | 定位报错行    | 数字 + 动作                 |
| 行内查找 | `f{字符}` | 向右找到该字符       | 一行内快速定位  | `f = find`              |
| 行内查找 | `t{字符}` | 跳到该字符前一位      | 精准停在目标前  | `t = till`              |
| 重复   | `;`     | 重复上次 `f/t` 方向 | 连续找同类符号  | 延续上次查找                  |
| 反向   | `,`     | 反向重复上次 `f/t`  | 返回前一个位置  | 与 `;` 配对记               |
| 重复修改 | `.`     | 重复上一次修改       | 批量重复小改动  | 一个点，表示再来一次              |

这部分不一定最先学，但在代码整理、格式修复时非常省时间。

## 命名规则

Vim 的命名并不是零散记忆，很多快捷键都有一套稳定结构。

### 1. 单字母多为动作或移动

| 键   | 含义         |
| --- | ---------- |
| `i` | insert，插入  |
| `a` | append，追加  |
| `d` | delete，删除  |
| `c` | change，修改  |
| `y` | yank，复制    |
| `p` | put，粘贴     |
| `u` | undo，撤销    |
| `w` | word，单词    |
| `b` | back，向后回单词 |
| `e` | end，单词末尾   |
| `f` | find，查找字符  |
| `t` | till，到某字符前 |

这一层只要知道英文缩写，大部分会自然联通。

---

### 2. 组合规则通常是“动作 + 范围”

这是 Vim 的核心语法。

| 结构       | 含义        | 例子         |
| -------- | --------- | ---------- |
| `d` + 范围 | 删除某段范围    | `dw` `d$`  |
| `c` + 范围 | 改写某段范围    | `cw` `c$`  |
| `y` + 范围 | 复制某段范围    | `yw`       |
| 数字 + 动作  | 执行多次或指定位置 | `3w` `10G` |

典型例子：

* `dw`：删除一个单词范围
* `c$`：改写到行尾
* `3w`：向前跳三个单词
* `5dd`：删除五行

看到组合时，不要拆成死记硬背的整体，而是按“前半部分做什么，后半部分作用到哪里”来理解。

---

### 3. 双击同键常表示整行或更大单位

| 快捷键  | 含义   |
| ---- | ---- |
| `dd` | 删除整行 |
| `cc` | 改整行  |
| `yy` | 复制整行 |
| `gg` | 到最开头 |

这一规律很实用。单个动作作用在局部，重复动作键常常扩大范围。

---

### 4. 大写版本常表示更强、更远、更大范围

| 小写  | 大写  | 差别                |
| --- | --- | ----------------- |
| `a` | `A` | 当前字符后插入 / 直接到行尾插入 |
| `i` | `I` | 当前字符前插入 / 直接到行首插入 |
| `o` | `O` | 下方开新行 / 上方开新行     |
| `v` | `V` | 字符选择 / 整行选择       |
| `g` | `G` | 配合跳转 / 直接到底部      |

可以把大写理解成“更大一级的版本”。

---

### 5. 文本对象规则是“动作 + inner/around + 对象”

这是代码编辑最有价值的一层。

| 结构                  | 含义          | 例子                |
| ------------------- | ----------- | ----------------- |
| `i`                 | inner，内部    | `ciw` `ci(` `ci"` |
| `a`                 | around，包含边界 | `da(` `da"`       |
| `w`                 | word，单词     | `diw`             |
| `(` `[` `{` `"` `'` | 对应括号、引号对象   | `ci{` `ci"`       |

理解后会非常清晰：

* `ciw`：改单词内部
* `ci(`：改括号内部
* `da"`：删掉整段引号内容，连引号一起删

## 记忆方法

### 1. 按语法记，不按清单记

比起死记 `dw`、`cw`、`yw` 三个命令，更有效的是记住：

* `d` 是删除
* `c` 是修改
* `y` 是复制
* `w` 是一个单词
* `$` 是到行尾

这样自然就能推出很多组合。

---

### 2. 按场景记

比起孤立背快捷键，更适合按真实动作分组。

#### 改一个词

* `ciw`
* `cw`

#### 清掉本行后半段

* `d$`
* `c$`

#### 改括号里的内容

* `ci(`

#### 改字符串内容

* `ci"`

#### 一行内跳到某个符号附近

* `f,`
* `f)`
* `t"`

这种记法更贴近使用过程，进入实战更快。

---

### 3. 把最常用动作练成反射

优先形成肌肉记忆的一组是：

* `w` `b` `0` `$`
* `i` `a` `o`
* `dd` `yy` `p`
* `/` `n`
* `.`

只要这组熟了，编辑速度就已经会明显变化。

---

### 4. 用“内外”去记文本对象

* `i` 是里面
* `a` 是连边框一起

例如：

* `ci"`：只改引号里面的内容，引号保留
* `da"`：整个带引号字符串一起删除

这个对比非常好记，因为动作边界很直观。

---

### 5. 用重复键和大写键的规律减少记忆负担

* 双键常是整行或更大单位：`dd` `yy` `cc`
* 大写常是更强版本：`A` `I` `V` `G`

这样不是一个个背，而是在已有模式上扩展。

---

## 最实用的一套记忆顺序

| 阶段   | 目标       | 快捷键                               |
| ---- | -------- | --------------------------------- |
| 第一阶段 | 能脱离方向键编辑 | `h j k l` `w b 0 $` `i a o`       |
| 第二阶段 | 能快速删改文本  | `dd` `yy` `p` `dw` `cw` `d$` `c$` |
| 第三阶段 | 能快速搜索与重复 | `/` `n` `f` `t` `;` `.`           |
| 第四阶段 | 能高效改代码结构 | `ciw` `ci(` `ci"` `V` `Ctrl-v`    |
| 第五阶段 | 能整理代码版式  | `>>` `<<` `==` `gg=G`             |

---

## 适合直接背下来的核心骨架

| 骨架                       | 含义                |
| ------------------------ | ----------------- |
| `动作 + 范围`                | `dw` `c$` `yw`    |
| `动作 + inner/around + 对象` | `ciw` `ci(` `da"` |
| `双击动作键`                  | `dd` `yy` `cc`    |
| `大写 = 更大范围`              | `A` `I` `V` `G`   |
| `.`                      | 重复上一次修改           |

掌握这一层之后，Vim 的快捷键不再像一堆零碎命令，而是一套可推导的编辑语法。这样记忆负担会小很多，迁移到新命令时也更快。🙂
